Browser Bridge Settings

All changes are saved instantly. Adjust security-related switches with care.

Security

Allow page_eval (execute arbitrary JavaScript)
When disabled, an AI call to page_eval is rejected outright and no code runs on the page. This is the most dangerous tool — even with a confirmation channel, disabling it completely eliminates the risk of silent code execution (ADR-0008).
Redact page_eval return values
Apply regex masking to eval results: JWTs, long hex, long numbers, and strings that look like tokens/keys (ADR-0008).
⚠ When disabled, tokens / cookies / keys may leak to the AI verbatim.
Confirm high-risk clicks (submit buttons / link navigation)
When a form submit button or link is clicked, a confirmation toast appears and requires user approval. This is the key line of defense against the AI silently placing orders / transferring funds / navigating away (ADR-0006).
⚠ When disabled, the AI's clicks on submit buttons/links are no longer confirmed. Only disable this if you fully trust the AI.
Confirm page_eval (execute arbitrary JS)
Every page_eval call shows a confirmation toast displaying the full code and requires user approval (ADR-0008). page_eval can execute arbitrary JavaScript and is the highest-risk operation.
⚠ When disabled, the AI executes arbitrary JS directly with no prompt. Only disable this if you fully trust the AI and need fully automated operation.
Confirm tab_close (close tab)
Before a tab is closed, an in-page "Close tab?" confirmation appears and requires user approval.
⚠ When disabled, the AI closes tabs directly with no prompt.
Warn before precise snapshot
Before page_snapshot_precise (which uses chrome.debugger) is called, show an informational prompt on the page. Chrome briefly displays a "Debugging" banner at the top (ADR-0009).

Tabs

Put AI-opened tabs into a "Browser Bridge" group
Tabs opened via tab_open are automatically placed into a named tab group, visually isolated from your own tabs, and can be collapsed/closed as a group. Enabled by default (ADR-0018).

Execution mode

Route all page operations through CDP (chrome.debugger)
When enabled, all page operations use CDP (chrome.debugger) instead of injecting content scripts; this allows page_eval to run on strict-CSP sites (such as Bing). Disabled by default (ADR-0017).
⚠ When enabled, Chrome persistently displays a "Debugging this browser" banner at the top (the debugger stays attached the whole time), and the access surface is larger. Only enable this when you need to bypass CSP or want unified deep control.

Tool enablement

For any unchecked tool, an AI call receives a "tool disabled in settings" error. Note: the AI can still see that the tool exists, but cannot execute it.

Allowed sites (allowlist)

Allow all sites (skip per-site approval)
When enabled, the AI can operate on any http(s) site directly, without a per-site authorization prompt for each new site. Enabling it requests the "<all_urls>" host permission. Suitable when you trust the AI to browse fully automatically; otherwise it's recommended to keep this off and use the per-site allowlist below.
⚠ When enabled, any site (including banking, email, and intranet sites) can be operated on without authorization. Use with care.
Format: https://domain/*. After adding manually, Chrome's host-permission prompt still needs to be approved on first visit to that site.
